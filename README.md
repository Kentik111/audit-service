# audit-service
Внутренний веб-сервис для работы с результатами аудитов безопасности финансовых систем

## Применяемые технологии
1. Python 3.10+ / Flask 3.0
2. SQLite + SQLAlchemy
3. Bootstrap 5 + Chart.js
4. Flask-Login (аутентификация)

## Реализованная функциональность
Сервис включает в себя:
1. ролевую модель (L1, L2, L3, Admin) с разграничением прав;
2. просмотр, фильтрацию и сортировку результатов аудита;
3. детальную карточку с комментариями;
4. дашборд с визуализацией статистики (4 графика);
5. калькуляторы риска, SLA и соответствия;
6. управление пользователями (доступно только администраторам).

## Инструкция по запуску
### 1. Склонировать репозиторий (через CMD, терминал или Git Bash)
```bash
git clone https://github.com/Kentik111/audit-service
cd audit-service
```
### 2. Создать виртуальное окружение
#### На Windows:
```bash
python -m venv venv
venv\Scripts\activate
```
#### На Linux / MacOS
```bash
python3 -m venv venv
source venv/bin/activate
```
### 3. Установить зависимости
```bash
pip install -r requirements.txt
```
### 4. Запустить приложение
```bash
python app.py
```
Приложение будет доступно по ссылке: http://127.0.0.1:5000
### 5. Остановить сервис
Остановка сервиса происходит сочетанием клавиш: Ctrl+C

## Тестовые учётные записи
При первом запуске автоматически создаются тестовые данные.
логин / пароль  
admin / admin - Администратор  
l1_user / l1 - Analyst L1  
l2_user / l2 - Analyst L2  
l3_user / l3 - Analyst L3  

## Ролевая модель
Analyst L1: просмотр списка, фильтрация, дашборд, калькуляторы  
Analyst L2: всё из L1, изменение статуса, добавление комментариев  
Analyst L3: всё из L2, изменение критичности, финальное решение  
Admin: полный доступ, управление пользователями  

## Примечания
1. База данных SQLite создаётся автоматически при первом запуске в папке instance/
2. При первом запуске создаётся 25 тестовых записей аудита
3. Для добавления новых пользователей используйте учётную запись admin или любую другую с правами администратора
